Rewrite test/backend-selection.test.ts to use the working mocking pattern from test/detection.test.ts.

The current issue: Tests set up spawnSync spy, then call jest.resetModules(), then require modules. The spy points to the OLD module, but after reset, a NEW module is loaded, so the mock doesn't intercept calls.

The fix:

1. Remove jest.resetModules() from beforeEach in "Backend Selection Logic" describe block
2. Set up spawnSync mock BEFORE requiring any modules in each test
3. Use mockReturnValue() for simple cases (like detection.test.ts does)
4. Use mockImplementation() only when different calls need different returns
5. Call jest.resetModules() AFTER setting up mocks but BEFORE requiring modules

Pattern to follow (from detection.test.ts lines 101-111):

```typescript
describe("Tool Availability Checking", () => {
  let spawnSyncMock: jest.SpyInstance;

  beforeEach(() => {
    // Mock spawnSync
    spawnSyncMock = jest.spyOn(require("child_process"), "spawnSync");
  });

  afterEach(() => {
    spawnSyncMock.mockRestore();
  });

  it("should return true when tool is installed", () => {
    spawnSyncMock.mockReturnValue({
      status: 0,
      stdout: "/usr/bin/xclip",
      stderr: "",
    });

    const { isToolAvailable } = require("../src/os");
    const result = isToolAvailable("xclip");

    expect(result).toBe(true);
  });
});
```
